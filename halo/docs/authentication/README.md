# Halo Authentication Methods

Currently supported authentication methods:

- Basic Auth
- Form Login

Planned:

- [Personal Access Token](https://github.com/halo-dev/halo/issues/1309)
- [OAuth2](https://oauth.net/2/)

## Basic Auth

The simplest authentication method. Set the HTTP header `Authorization: Basic xxxyyyzzz==` to authenticate with the Halo API:

```bash
╰─❯ curl -u "admin:P@88w0rd" -H "Accept: application/json" http://localhost:8090/api/v1alpha1/users

# Or
╰─❯ echo -n "admin:P@88w0rd" | base64
YWRtaW46UEA4OHcwcmQ=
╰─❯ curl -H "Authorization: Basic YWRtaW46UEA4OHcwcmQ=" -H "Accept: application/json" http://localhost:8090/api/v1alpha1/users
```

## Form Login

A common authentication method requiring username, password, and a CSRF token (to prevent duplicate submissions and cross-site request forgery).

- Form Parameters

  | Parameter  | Type   | Description                                             |
  |------------|--------|---------------------------------------------------------|
  | username   | form   | Username                                                |
  | password   | form   | Password                                                |
  | _csrf      | form   | CSRF token, randomly generated by the client            |
  | XSRF-TOKEN | cookie | Cross-site request forgery token, must match `_csrf`    |

- HTTP 200 Response

  Only returned when the `Accept` header contains `application/json`:

  ```bash
  ╰─❯ curl 'http://localhost:8090/login' \
    -H 'Accept: application/json' \
    -H 'Cookie: XSRF-TOKEN=1ff67e0c-6f2c-4cf9-afb5-81bc1015b8e5' \
    -H 'Content-Type: application/x-www-form-urlencoded' \
    --data-raw '_csrf=1ff67e0c-6f2c-4cf9-afb5-81bc1015b8e5&username=admin&password=P@88w0rd'
  ```


## Personal Access Token

### Background

Halo is a modern open-source CMS / website building tool. To make it easier for developers and users to access site data via the API, Halo supports Personal Access Tokens (PAT).
Users can generate a PAT in the Halo admin panel — it is a random string used for identity verification in API request headers. The Halo backend validates the PAT value on each request and grants access to the corresponding API data if it matches.
This PAT mechanism avoids the security risks of using raw username/password credentials. Developers can set access scopes, expiration times, and other constraints for each PAT. Using random PATs improves security and provides a safer, more standardized authentication method for Halo plugins and applications.

### Design

### Usage

#### Generating a PAT

Halo provides a dedicated endpoint: `/apis/api.console.security.halo.run/v1alpha1/users/-/personalaccesstokens`.

```shell
curl -u admin:admin -X 'POST' \
  'http://localhost:8090/apis/api.console.security.halo.run/v1alpha1/users/-/personalaccesstokens' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
  "spec": {
    "name": "My PAT",
    "description": "This is my first PAT.",
    "expiresAt": "2023-09-15T02:42:35.136Z"
    "roles": [""]
  }
}'
```

Request body fields:

| Property    | Description                                                                                        |
|-------------|----------------------------------------------------------------------------------------------------|
| name        | PAT name. Required.                                                                                |
| description | PAT description. Optional.                                                                         |
| expiresAt   | PAT expiration time. Cannot be modified after creation. If omitted, the PAT never expires.         |
| roles       | Roles granted to the PAT. Must be a subset of the current user's roles. If `null` or `[]`, the PAT only has `anonymous` and `authenticated` roles. |

Response fields:

| Field Path                  | Description                                                                          |
|-----------------------------|--------------------------------------------------------------------------------------|
| security.halo.run/access-token | The generated PAT. This value is not stored in the database, so it can only be saved once. |
